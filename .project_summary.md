## ­¤Éø Critical Bug Fixes ŌĆö 2026-03-05

### ŌÜĀ’ĖÅ IMPORTANT FOR FUTURE AGENTS ŌĆö DO NOT REPEAT THESE MISTAKES

#### Bug 1: Workgroup File Upload Crash
- **File**: `app/Filament/Workgroup/Pages/SharedUploads.php`
- **Error**: `Call to a member function store() on string`
- **Cause**: Filament's `FileUpload` component in Action forms returns a **string** (temporary Livewire file path), NOT an `UploadedFile` object. Calling `$file->store()` on a string crashes.
- **Fix**: Detect file type: if string, use `Storage::disk('local')->get()` then `Storage::disk('public')->put()` to move it. If `UploadedFile`, use standard `$file->store()`.
- **Pattern**: Always check `is_string($file)` vs `$file instanceof UploadedFile` when handling Filament FileUpload data.

#### Bug 2: Workgroup Panel 403 for Admin Users
- **File**: `app/Models/User.php` + `app/Http/Middleware/EnsureWorkgroupPanelAccess.php`
- **Error**: 403 Forbidden when admin users tried to access `/workgroups`
- **Cause**: `canAccessPanel('workgroups')` only allowed `super_admin`, `workgroup_*` roles ŌĆö missing `admin` and `logistics_admin`.
- **Fix**: Added `admin` and `logistics_admin` to both `User::canAccessPanel()` and `EnsureWorkgroupPanelAccess` middleware.

#### Bug 3: Notes Page 500 Error for All Users
- **File**: `app/Models/WorkgroupNote.php`
- **Error**: `Argument #1 ($length) must be of type int, null given`
- **Cause**: `getPreviewAttribute(int $length = 100)` had `int` type hint. Laravel's Eloquent accessor system passes `null` as the first argument when calling accessors, which PHP rejects when typed `int`.
- **Fix**: Changed to `getPreviewAttribute($length = null)` and use `(int) ($length ?? 100)` inside.
- **Pattern**: NEVER use `int` type hint on Eloquent accessor methods ŌĆö Eloquent passes `$value` (null for computed attributes) as first arg.

#### Bug 4: Evaluation Form 500 on Re-visit
- **File**: `app/Services/Workgroup/EvaluationService.php` ŌåÆ `getOrCreateDraft()`
- **Error**: `SQLSTATE[23505]: Unique violation: duplicate key value violates unique constraint "evaluation_submissions_workgroup_member_id_candidate_product_id"`
- **Cause**: `getOrCreateDraft()` filtered by `status = 'draft'`. When a user had already **submitted** (status = 'submitted'), it couldn't find the existing submission and tried to INSERT a new one, violating the unique constraint.
- **Fix**: Removed the `->where('status', 'draft')` filter so it finds ANY existing submission for the member+product combination.

#### Bug 5: Evaluation Form Read-Only Lock (User Request Fix)
- **File**: `app/Filament/Workgroup/Pages/EvaluationFormPage.php`
- **Issue**: Submitted evaluations were shown as read-only, users couldn't edit.
- **Fix**: Removed `$this->isReadOnly = true` from both the `loadProduct()` method and the `submitEvaluation()` method. Evaluations are now always editable.

### Root Cause of 500 Error
`Filament\Actions\SelectAction` does not exist in Filament v3. The class was imported and used in `SessionResultsPage.php`, causing a fatal error on every page load.

1. **SelectAction doesn't exist** ŌĆö Replaced with `Action` + `Select` form (Filament v3 pattern)
2. **FinalistsWidget uses BadgeColumn** ŌĆö Replaced deprecated Filament v2 `BadgeColumn` with `TextColumn::badge()`
