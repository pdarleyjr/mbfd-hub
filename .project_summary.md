## What Is MBFD Hub?

MBFD Hub is the Miami Beach Fire Department's internal operations platform — a containerized Laravel application running on a dedicated VPS at `145.223.73.170`.

It currently provides:
- Daily operational forms
- Vehicle inspections and defect reporting
- Station inventory workflows
- Capital project tracking
- Workgroup / evaluation management
- Training resources
- Personnel, apparatus, and equipment administration
- Google Sheets apparatus synchronization
- Equipment intake and Snipe-IT integration

## Major Implemented Systems

### 1. Daily Vehicle Inspections
- Public React flow for apparatus inspection submission
- Vehicle selection by active designation while preserving canonical vehicle number
- Checklist normalization for React consumption
- Historical inspection viewing in Filament
- Digital signature capture
- HOLD / out-of-service automation for critical defects

### 2. Station Inventory V2
- PIN-gated station access
- On-hand count workflow with PAR threshold awareness
- Special supply request path
- Audit trail with employee / shift tracking

### 3. Workgroup / Eval Feedback Hub
- Third Filament panel at `/workgroups`
- Evaluation templates, candidate products, sessions, feedback, rankings, finalists
- Exporters for scores, finalists, feedback, and completion status
- Workgroup analytics dashboard and AI-assisted report flows
- **Brand aggregation and competitor grouping** via EvaluationService
- **SAVER document generator** — AI-powered purchasing reports
- Widget-free architecture (all data inlined via `getViewData()`)

### 4. Google Sheets Apparatus Sync
- One-way sync from Fire Apparatus admin data to Google Sheets
- Job-based dispatch after apparatus updates
- Manual recovery command via `artisan apparatus:sync-sheet`

### 5. Equipment Intake + Snipe-IT
- Admin equipment intake page
- AI-assisted photo extraction flow
- Bulk/manual asset entry path
- Snipe-IT service integration and SAML groundwork

### 6. Chatify / Reverb Real-Time Messaging
- Public/frontend connection uses `www.mbfdhub.com` over WSS
- Backend broadcasting uses internal Reverb endpoint
- Queue worker + Reverb run inside the Laravel container

### 7. Pump Simulator V2
- Standalone React training simulator
- Zustand-based state management
- Multi-line pump panel and nozzle modeling
- No `@apply` usage in simulator CSS

### 8. Enterprise Modernization (2026-03-11)
- **Laravel Pulse** real-time dashboard at `/admin/pulse` for request throughput, slow queries, exceptions, queue health
- **Spatie Health** checks registered via `HealthServiceProvider` (database, cache, queue, disk, env)
- **Cloudflare AI Gateway** routing for vision-agent and workgroup-ai Workers (caching layer, analytics, rate limiting)
- **PWA hardening** with Dexie IndexedDB for offline form drafts, React Query for sync, vite-plugin-pwa for Workbox service worker
- **Fire Equipment Requests** and **Station Inspections** — new schema, API controllers, Filament resources
- **FormsHub wizards** — multi-step Equipment Request and Station Inspection forms in the React SPA
- **Filament admin restructuring** — Station Management navigation group with EquipmentRequests and StationInspections relation managers

### 9. Notification System
- **Web Push Notifications**: Browser push via `laravel-notification-channels/webpush` with VAPID keys. Used for real-time chat message alerts (`ChatMessageReceived`) with rate limiting.
- **In-App Database Notifications**: Filament native notifications polled every 30s. Used for project management alerts (overdue projects, milestones, budget alerts) triggered by scheduled console commands, and UI action feedback.
- **Notification Preferences** (2026-03-12): `notification_preferences` JSON column on users table. `NotificationSettings` Filament page with 5 toggle categories accessible from Admin and Workgroup user menus. `AppServiceProvider` filters recipients by preferences before dispatch. Station inventory submissions now trigger `NewSubmissionNotification`.
- **WebPush Debug Logging** (2026-03-12): `NewSubmissionNotification` implements `ShouldQueue` with `failed()` logging. `PushSubscriptionController` logs store/delete with payload diagnostics. `push-notification-widget.js` logs subscription payload and server responses.
- **Service Workers**: `public/sw.js` handles push events for main panels. ⚠️ **ISSUE**: `public/daily/sw.js` (React SPA) currently lacks the push event listener due to Vite build overwrites (see ERROR-036).
