## What Is MBFD Hub?

MBFD Hub is the Miami Beach Fire Department's internal operations platform — a containerized Laravel application running on a dedicated VPS at `145.223.73.170`.

It currently provides:
- Daily operational forms
- Vehicle inspections and defect reporting
- Station inventory workflows
- Capital project tracking
- Workgroup / evaluation management
- Training resources
- Personnel, apparatus, and equipment administration
- Google Sheets apparatus synchronization
- Equipment intake and Snipe-IT integration

## Major Implemented Systems

### 1. Daily Vehicle Inspections
- Public React flow for apparatus inspection submission
- Vehicle selection by active designation while preserving canonical vehicle number
- Checklist normalization for React consumption
- Historical inspection viewing in Filament
- Digital signature capture
- HOLD / out-of-service automation for critical defects

### 2. Station Inventory V2
- PIN-gated station access
- On-hand count workflow with PAR threshold awareness
- Special supply request path
- Audit trail with employee / shift tracking

### 3. Workgroup / Eval Feedback Hub
- Third Filament panel at `/workgroups`
- Evaluation templates, candidate products, sessions, feedback, rankings, finalists
- Exporters for scores, finalists, feedback, and completion status
- Workgroup analytics dashboard and AI-assisted report flows

### 4. Google Sheets Apparatus Sync
- One-way sync from Fire Apparatus admin data to Google Sheets
- Job-based dispatch after apparatus updates
- Manual recovery command via `artisan apparatus:sync-sheet`

### 5. Equipment Intake + Snipe-IT
- Admin equipment intake page
- AI-assisted photo extraction flow
- Bulk/manual asset entry path
- Snipe-IT service integration and SAML groundwork

### 6. Chatify / Reverb Real-Time Messaging
- Public/frontend connection uses `www.mbfdhub.com` over WSS
- Backend broadcasting uses internal Reverb endpoint
- Queue worker + Reverb run inside the Laravel container

### 7. Pump Simulator V2
- Standalone React training simulator
- Zustand-based state management
- Multi-line pump panel and nozzle modeling
- No `@apply` usage in simulator CSS
